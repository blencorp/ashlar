# v0.1 — Public alpha (12 weeks, cumulative 20)

The first public release. Open-source repository, public registry, public docs site. Goal: validate the architecture beyond design partners; recruit ~10 broader-adopter teams; harden the drift, AI, and validation systems against real-world usage.

## Goal in one sentence

Demonstrate that Ashlar addresses shadcn's #1 production complaint (drift) and shadcn's biggest AI-integration gap (no validate/migrate MCP) at public-alpha scale, while expanding component coverage and shipping the first L3 pattern.

## Deliverables

### Component coverage

Add seven more L0 components and three more L1 components:

**L0 additions:**
13. Tabs (using ARIA + JS-free where possible; details/summary fallback for non-JS environments)
14. Pagination
15. Tag / Badge
16. Skeleton / Loading state
17. Tooltip (using popover API + anchor positioning)
18. Avatar
19. Stepper / Step indicator

**L1 additions:**
20. Toast / Notification stack
21. Combobox v2 (filtering + async loading enhancement on top of v0.0 base)
22. Date Picker (single date with native fallback; range deferred to v0.2)

Cumulative v0.1 components: **22** (12 from v0.0 + 7 new L0 + 3 new L1).

### First L3 pattern

- **Address form** pattern (composed of FormField, TextInput, autocomplete-aware select for state/country).

Includes: composition, content guidance for plain language, accessibility evidence, codemods, multi-template renderings (Nunjucks + Twig + Jinja + ERB + plain HTML).

### MCP server

Ship `npx ashlar mcp` with the full tool set:

- `search_components`, `get_component`, `validate_usage`, `suggest_for_task`, `migrate`, `get_evidence`, `list_tokens`, `get_token`.
- Resources: `capsule://`, `token://`, `pattern://`, `evidence://`, `docs://`.
- Prompts: `build-form`, `apply-agency-theme`, `migrate-from-react-uswds` (skeleton).
- Local stdio mode + optional hosted SSE mode.

### Codemods

- Codemod ship-with-capsule infrastructure proven via 5+ real codemods (covering version transitions for Button, FormField, Alert, Dialog, Combobox).
- ast-grep rule generation from CEM `_ashlar.antiPatterns` fully automated.

### Drift management

- Tighten conflict rate target to <10%.
- Telemetry (opt-in) on update outcomes; instrument for the 100+ updates expected during alpha.
- `ashlar update --resolved` flow polished.
- Force-confirmation UX for accessibility-critical changes refined.

### Distribution

- Public registry at `https://registry.ashlar.dev` with CDN fronting.
- Signed Git mirror at `https://github.com/blencorp/ashlar-registry`.
- Sigstore signing fully wired via GitHub Actions (keyless OIDC).
- Public docs site at `https://ashlar.dev` with llms.txt + llms-full.txt.

### Templates

L4 template targets: Nunjucks, Twig, Jinja, ERB, plain HTML for all 22 components. Snapshot equality tests enforce DOM-contract parity.

### Accessibility evidence

- All 22 components reach at least `automated-tested`.
- At least 12 components reach `stable-evidence` (full NVDA + VoiceOver + JAWS manual tests, evidence packet complete).
- Independent accessibility consultant review of 5 components.

### Demo apps

- Original 3 from v0.0, expanded.
- New: Vite SPA demo with Combobox / async filtering.
- New: Drupal site with the address-form pattern in production-style configuration.

## Cumulative success criteria

- [ ] All v0.0 success criteria still pass at v0.1 scale.
- [ ] Drift conflict rate <10% across 100+ updates.
- [ ] MCP server validated by ≥3 AI editors (Cursor, Claude Code, Windsurf) with zero hallucinated props in eval suite.
- [ ] At least 5 design partner teams running Ashlar in non-toy projects.
- [ ] At least 12 components at `stable-evidence`.
- [ ] L3 address-form pattern works in all 5 template languages with snapshot parity.
- [ ] Bundle size still meets targets (<10KB L0-only for 5 components, <15KB with one L1).

## Public-alpha communications

v0.1 is the first public moment. Important to set expectations:

- **README + landing page**: clear "alpha" labeling. "Ready to evaluate; not ready for stable production."
- **Comparison docs**: explicit comparison to USWDS, shadcn, Carbon — what overlaps, what differs, what we don't yet do.
- **Migration docs**: starting points for `react-uswds` and shadcn migration.
- **Contribution guide**: maintainer charter, RFC process, accessibility-evidence gates.
- **Public roadmap**: this document, plus monthly community calls.

## Risks specific to v0.1

- **Public scrutiny exposes architectural gaps.** Mitigation: invite skeptical reviewers (USWDS contributors, accessibility specialists, gov-tech leaders) to private review before public alpha.
- **Drift conflict rate higher than expected at scale.** Mitigation: telemetry to spot the patterns; ship conflict-resolution UX improvements within first month of v0.1.
- **MCP server gets misused.** Mitigation: read-only by default; explicit confirmation gates on `add_component`/`update_component`.
- **Maintainer bandwidth.** Mitigation: enforce multi-org charter from day 1; do not allow Ashlar to become bus-factor=1 (the AuDS lesson).

## What v0.1 unlocks

- Public adoption signal (stars, downloads, contributors).
- Real-world drift data to refine merge UX.
- AI-integration validation at scale.
- L3 pattern infrastructure proven; ready for v0.2 expansion.
- Foundation for multi-framework expansion (v0.2).
