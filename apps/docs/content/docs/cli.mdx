---
title: CLI
description: Use the Ashlar CLI to initialize an app, add source, and verify it.
---

Run the CLI from the app that should own the copied source. Most teams only need
three commands to start.

## Start here

<Steps>
<Step>

### Initialize Ashlar

Creates `ashlar.config.json`, `ashlar-lock.json`, generated theme files, and the
default source directory.

<AshlarCommand args="init" />

</Step>
<Step>

### Add components

Copies source files into your app. Start small, then add more components as the
screen needs them.

<AshlarCommand args="add button alert text-input" />

</Step>
<Step>

### Verify the install

Checks the installed files against the lockfile and bundled registry metadata.

<AshlarCommand args="verify" />

</Step>
</Steps>

## Package managers

Every command block has npm, pnpm, and bun tabs. Use the package manager your app
already uses.

## Find components

Use search when you know the component name. Use suggest when you know the task
but not the component list.

<AshlarCommand lines={["search button", 'suggest "Build a renewal case board"']} />

## Preview changes

Use dry runs before writing files or updating installed source.

<AshlarCommand lines={["add button --dry-run --diff", "update button --dry-run"]} />

## Audit before release

<AshlarCommand args={'audit --policy all "src/**/*.{html,tsx,jsx,vue,svelte}"'} />

Use `--policy components` for installed-component rules or `--policy all` for
the broadest local check.
