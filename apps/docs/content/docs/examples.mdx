Examples should prove that Ashlar works inside real app shapes. Each example
should start from the same user flow: initialize, add components, import CSS,
verify.

## Case-board demo

The current product demo is the Vite benefits case board. It is a dense
public-service workflow, not a landing page.

```bash
pnpm --filter @blen/ashlar-example-vite dev
```

Look for:

- official website banner and identifier footer;
- queue metrics, filters, status messaging, and case actions;
- Default, VA, and USDA theme switching;
- light, dark, and system modes;
- mobile layout with the same content and controls.

## Standard flow

<AshlarCommand lines={["init", "add button alert text-input", "verify"]} />

## Framework matrix

- Vanilla: import the generated stylesheet from your main CSS file and use the
  copied HTML/CSS contracts directly.
- React SPA: import the generated stylesheet from `src/main.tsx` or your global
  CSS, then wrap copied markup in your app components.
- Next.js: import the generated stylesheet from `app/globals.css`.
- Svelte: import the generated stylesheet from your app-level CSS or layout.
- Vue: import the generated stylesheet from `src/main.ts` or your global CSS.

## Example quality bar

- Use real public-service workflows like case queues, renewal forms, document
  review, and approvals.
- Cover vanilla, React SPA, Next.js, Svelte, Vue, and Vite with the same
  init/add/verify path.
- Exercise theming, forms, alerts, buttons, identifiers, and validation states.
- Include install, build, audit, and visual smoke coverage before calling an
  example representative.
