Ashlar components are source capsules: implementation files, CEM metadata,
evidence records, hashes, and update metadata. Use the CLI to copy them into your
app and verify the installed source after changes.

## Add a single capsule

```bash
pnpm dlx @blen/ashlar@latest search button
pnpm dlx @blen/ashlar@latest add button --dry-run --diff
pnpm dlx @blen/ashlar@latest add button
pnpm dlx @blen/ashlar@latest verify
```

## Add a form path

```bash
pnpm dlx @blen/ashlar@latest add \
  form-field text-input textarea date-input select \
  radio-group checkbox button alert error-summary identifier
```

## Integration rules

- Do not nest cards to make component pages look busier.
- Keep examples task-oriented: intake forms, case queues, document review, and
  service-flow confirmation.
- Component docs must include copyable install commands, source paths, evidence
  status, and known gaps.
- Claim stability only from evidence records, not from visual polish.
