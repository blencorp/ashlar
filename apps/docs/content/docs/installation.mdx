title: Installation

---

Install Ashlar in the application you are building. The CLI package is
`@blen/ashlar`.

## 1. Initialize

<AshlarCommand args="init" />

`init` writes:

- `ashlar.config.json` for registry, output paths, and style paths.
- `ashlar-lock.json` for installed component hashes.
- `src/ashlar/ashlar.css` plus generated theme files.
- `AGENTS.md` and `DESIGN.md` context files for AI-assisted work.

## 2. Add components

<AshlarCommand args="add button alert text-input" />

Preview a write before changing files:

<AshlarCommand args="add button --dry-run --diff" />

## 3. Import styles

Import the generated stylesheet from your app's global stylesheet or entry file.
The path depends on your framework:

```css
@import "./src/ashlar/ashlar.css";
```

For Next.js this usually lives in `app/globals.css`. For Vite, Vue, and Svelte
apps it usually belongs in the root stylesheet imported by the app entry.

## 4. Verify

<AshlarCommand args="verify" />

Run `verify` after installing, updating, or editing Ashlar-managed files.
