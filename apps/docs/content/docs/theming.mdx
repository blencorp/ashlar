Ashlar ships a `default` theme and source-derived agency adapters. The adapters
are token files grounded in public guidance; they are not agency endorsements or
official brand systems.

## Sync generated theme output

<AshlarCommand lines={["theme sync", "theme validate"]} />

## Theme names

- `default`: Ashlar's base public-service theme.
- `va`: source-derived adapter based on VA.gov Design System color and
  typography guidance.
- `usda`: source-derived adapter based on USDA digital brand guidance and USWDS
  color guidance.

## Recorded sources

- USWDS color guidance:
  https://designsystem.digital.gov/design-tokens/color/overview/
- USWDS system color tokens:
  https://designsystem.digital.gov/design-tokens/color/system-tokens/
- VA.gov color palette:
  https://design.va.gov/foundation/color-palette
- VA.gov typography:
  https://design.va.gov/foundation/typography
- USDA Design and Brand Plays:
  https://www.usda.gov/about-usda/policies-and-links/digital/digital-strategy/design-and-brand/design-and-brand-plays

## Review checklist

- Forced-colors and visible focus must survive theme switching.
- Token names should describe intent, not agency color trivia.
- Agency logos belong in agency selectors and examples only when usage rights and
  source guidance are explicit.
- Visual tests should cover light, dark, and system mode for every theme.
