Ashlar installs public-service UI source into your application. You run the CLI,
choose capsules, and own the copied files in your repo.

The model is deliberately close to shadcn's source-owned workflow, but the proof
layer is different: hashes, lockfiles, update survival, public-service audits,
theme provenance, AI-readable contracts, and release gates.

<Callout type="warn" title="Current release posture">
Ashlar is early software. Use it as source-owned starter material, not as an
accessibility-stable replacement for an agency design system.
</Callout>

## What to test first

From this checkout, the most useful product test is the Vite case board:

```bash
pnpm --filter @blen/ashlar-example-vite dev
```

Open the local URL and check whether the same components hold up across the
Default, VA, and USDA themes, plus light, dark, and system modes.

For a clean consumer install after public npm publish, the intended first-run
path is:

<AshlarCommand lines={["init", "add button alert text-input", "verify"]} />

## The model

<Steps>
<Step>

### Initialize

`init` writes `ashlar.config.json`, `ashlar-lock.json`, generated theme files,
and AI-readable project guidance.

</Step>
<Step>

### Add source

`add` copies selected components into your configured source directory. The
default path is `src/ashlar/components`.

</Step>
<Step>

### Verify

`verify` compares installed files with the lockfile and bundled registry
metadata so local changes are visible.

</Step>
</Steps>

## Principles

- Source ownership: installed files live in the consuming app.
- Small runtime surface: the CLI does the registry work; the app keeps ordinary
  source and CSS.
- Evidence before claims: accessibility and agency alignment must be backed by
  review records, tests, and cited sources.
- Default first: the base theme is named `default`. Agency themes are explicit
  adapters, not endorsements.

## How Ashlar should feel

- Like shadcn when installing source.
- Like a government validator when checking public-service rules.
- Like a token compiler when producing global CSS.
- Like a release engineer when proving provenance and evidence.
